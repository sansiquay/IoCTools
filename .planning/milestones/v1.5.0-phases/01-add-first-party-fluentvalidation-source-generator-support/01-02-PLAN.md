    2. Add the partial method call before `return services;` in the generated method body. Insert:
       `Add{methodNamePrefix}FluentValidationServices(services);`
       where `{methodNamePrefix}` uses the same `methodNamePrefix` parameter passed to the method.
       Place this AFTER all registration code and BEFORE `return services;`.

    3. Add the partial method declaration AFTER the main method, inside the class body:
       `static partial void Add{methodNamePrefix}FluentValidationServices(IServiceCollection services);`

    The modified template (around line 176) should become:
    ```
    return $$"""
             #nullable enable
             namespace {{extNameSpace}};

             {{usings.ToString().Trim()}}

             public static partial class GeneratedServiceCollectionExtensions
             {
                 public static IServiceCollection Add{{methodNamePrefix}}RegisteredServices(this IServiceCollection services{{configParameter}})
                 {
                      {{registrations.ToString().Trim()}}
                      Add{{methodNamePrefix}}FluentValidationServices(services);
                      return services;
                 }

                 static partial void Add{{methodNamePrefix}}FluentValidationServices(IServiceCollection services);
             }
             """;
    ```

    CRITICAL: The partial method call uses the SAME `methodNamePrefix` variable that the main method name uses. This ensures both generators derive the same method name from the same assembly name input.

    CRITICAL: `static partial void` with no access modifier is valid in ALL C# versions that support partial methods (C# 3.0+). Unimplemented partial methods are silently removed by the compiler. This is the safe approach per research pitfall analysis.
