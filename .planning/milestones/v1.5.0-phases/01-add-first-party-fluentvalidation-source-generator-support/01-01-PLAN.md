  <action>
    Create the core model and utilities following established patterns.

    **ValidatorClassInfo.cs:**
    - Namespace: `IoCTools.FluentValidation.Models`
    - `internal readonly struct ValidatorClassInfo : IEquatable<ValidatorClassInfo>`
    - Properties (all get-only):
      - `INamedTypeSymbol ClassSymbol` — the validator class
      - `TypeDeclarationSyntax ClassDeclaration` — the syntax node
      - `SemanticModel SemanticModel` — for later analysis
      - `INamedTypeSymbol ValidatedType` — the T from AbstractValidator<T>
      - `string Lifetime` — "Scoped", "Singleton", or "Transient" (from IoCTools attribute)
      - `string FullyQualifiedName` — cached ClassSymbol.ToDisplayString()
      - `string ValidatedTypeFullyQualifiedName` — cached ValidatedType.ToDisplayString()
    - Constructor taking all parameters
    - Manual `Equals(ValidatorClassInfo other)`: compare `FullyQualifiedName` and `ValidatedTypeFullyQualifiedName` (string equality for incremental pipeline caching)
    - Manual `GetHashCode()`: use unchecked multiply-add pattern on FullyQualifiedName.GetHashCode() and ValidatedTypeFullyQualifiedName.GetHashCode(). Do NOT use HashCode.Combine (not in netstandard2.0).
    - `Equals(object obj)` override, `==` and `!=` operators
    - Nullable `ValidatorClassInfo?` return from pipeline (for filtering)

    **FluentValidationTypeChecker.cs:**
    - Namespace: `IoCTools.FluentValidation.Utilities`
    - `internal static class FluentValidationTypeChecker`
    - Method `GetAbstractValidatorTypeArgument(INamedTypeSymbol classSymbol)`:
      - Walk BaseType chain until System.Object
      - Match by name: `current.Name == "AbstractValidator"` AND `current.ContainingNamespace?.ToDisplayString() == "FluentValidation"` AND `current.IsGenericType` AND `current.TypeArguments.Length == 1`
      - Return `current.TypeArguments[0] as INamedTypeSymbol` on match, null otherwise
    - Method `IsAbstractValidator(INamedTypeSymbol classSymbol)`:
      - Returns `GetAbstractValidatorTypeArgument(classSymbol) != null`
    - Method `GetLifetimeFromAttributes(INamedTypeSymbol classSymbol)`:
      - Check for ScopedAttribute, SingletonAttribute, TransientAttribute by name
      - Attribute names: "ScopedAttribute", "SingletonAttribute", "TransientAttribute" in namespace "IoCTools.Abstractions.Annotations"
      - Return "Scoped", "Singleton", "Transient", or null if no lifetime attribute
    - Method `HasLifetimeAttribute(INamedTypeSymbol classSymbol)`:
      - Returns `GetLifetimeFromAttributes(classSymbol) != null`
    - Method `IsValidatorInterface(INamedTypeSymbol interfaceSymbol)`:
      - Returns true if `interfaceSymbol.Name == "IValidator"` AND `interfaceSymbol.ContainingNamespace?.ToDisplayString() == "FluentValidation"` AND `interfaceSymbol.IsGenericType` AND `interfaceSymbol.TypeArguments.Length == 1`
      - This is for D-08 registration refinement

    **FluentValidationDiagnosticDescriptors.cs:**
    - Namespace: `IoCTools.FluentValidation.Diagnostics`
    - `internal static class FluentValidationDiagnosticDescriptors`
    - Start at IOC100 per research recommendation
    - Define stub descriptors (category "IoCTools.FluentValidation"):
      - IOC100: "Validator directly instantiates DI-managed child validator" (Warning) — for D-13
      - IOC101: "Validator composition creates lifetime mismatch" (Warning) — for D-12 lifetime propagation
      - IOC102: "Validator class missing partial modifier" (Error) — validator-specific reminder (IOC080 covers general case, but this gives FV-specific guidance)
    - Each descriptor: `new DiagnosticDescriptor(id, title, messageFormat, category, defaultSeverity, isEnabledByDefault: true, helpLinkUri: "https://github.com/nate123456/IoCTools/blob/main/docs/diagnostics.md#iocXXX")`
