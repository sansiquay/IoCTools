  <action>
    **FluentValidationFixtureHelper.cs:**
    - Namespace: `IoCTools.Testing.CodeGeneration`
    - `internal static class FluentValidationFixtureHelper`

    - Method `IsFluentValidatorType(ITypeSymbol typeSymbol)` returning `bool`:
      - Check if type is `IValidator<T>` by name: `typeSymbol.Name == "IValidator"` AND `typeSymbol.ContainingNamespace?.ToDisplayString() == "FluentValidation"` AND generic with 1 type argument
      - Return true if match

    - Method `HasFluentValidationReference(Compilation compilation)` returning `bool`:
      - Check `compilation.ReferencedAssemblyNames.Any(a => a.Name == "FluentValidation")`
      - This implements D-17: only generate helpers when FluentValidation is in references

    - Method `GenerateSetupHelpers(string mockFieldName, string validatedTypeName, string parameterName)` returning `string`:
      - Generate two methods:
        ```csharp
        protected void Setup{ParameterName}ValidationSuccess()
        {
            {mockFieldName}.Setup(v => v.Validate(It.IsAny<{validatedTypeName}>()))
                .Returns(new FluentValidation.Results.ValidationResult());
            {mockFieldName}.Setup(v => v.ValidateAsync(It.IsAny<{validatedTypeName}>(), It.IsAny<CancellationToken>()))
                .ReturnsAsync(new FluentValidation.Results.ValidationResult());
        }

        protected void Setup{ParameterName}ValidationFailure(params string[] errorMessages)
        {
            var failures = errorMessages.Select(m => new FluentValidation.Results.ValidationFailure("", m)).ToList();
            var result = new FluentValidation.Results.ValidationResult(failures);
            {mockFieldName}.Setup(v => v.Validate(It.IsAny<{validatedTypeName}>()))
                .Returns(result);
            {mockFieldName}.Setup(v => v.ValidateAsync(It.IsAny<{validatedTypeName}>(), It.IsAny<CancellationToken>()))
                .ReturnsAsync(result);
        }
        ```
      - Use fully-qualified FluentValidation types to avoid using statement requirements
      - The `parameterName` is PascalCased for method naming (e.g., `orderValidator` -> `OrderValidator` -> `SetupOrderValidatorValidationSuccess`)

    **FixtureEmitter.cs update:**
    - In the section where constructor parameters are processed and mock fields are generated:
      1. After generating the standard `Mock<IValidator<T>>` field, check if `FluentValidationFixtureHelper.IsFluentValidatorType(paramType)` AND `FluentValidationFixtureHelper.HasFluentValidationReference(compilation)`
      2. If both true, extract the validated type name from the generic type argument: `((INamedTypeSymbol)paramType).TypeArguments[0].ToDisplayString()`
      3. Call `FluentValidationFixtureHelper.GenerateSetupHelpers(mockFieldName, validatedTypeName, parameterName)` and append the result to the generated fixture class body
    - The Compilation reference is needed in the emitter. Check if it's already available in the Emit method context. If not, thread it through from the pipeline.
    - IMPORTANT: Do NOT add FluentValidation as a PackageReference to IoCTools.Testing. The helper detects types by name only, same pattern as the generator (per D-17).
