The main generator now emits:
```csharp
public static partial class GeneratedServiceCollectionExtensions
{
    public static IServiceCollection Add{Prefix}RegisteredServices(...)
    {
        // ... existing registrations ...
        Add{Prefix}FluentValidationServices(services);
        return services;
    }
    static partial void Add{Prefix}FluentValidationServices(IServiceCollection services);
}
```

The FV generator must emit the implementation of this partial method in a separate .g.cs file.
The namespace must be `{safeRootNamespace}.Extensions.Generated` where safeRootNamespace = assemblyName with `-` and ` ` replaced by `_`.
The class name must be `GeneratedServiceCollectionExtensions`.

    **ValidatorPipeline.cs:**
    - Namespace: `IoCTools.FluentValidation.Generator.Pipeline`
    - `internal static class ValidatorPipeline`
    - Method `Build(IncrementalGeneratorInitializationContext context)` returning `IncrementalValuesProvider<ValidatorClassInfo>`
    - Use `context.SyntaxProvider.CreateSyntaxProvider`:
      - Predicate: `node is TypeDeclarationSyntax` (same as ServiceClassPipeline)
      - Transform:
        1. Get `INamedTypeSymbol` from semantic model
        2. Skip if null, abstract, static, or not a class
        3. Call `FluentValidationTypeChecker.HasLifetimeAttribute(symbol)` -- skip if false (per D-07)
        4. Call `FluentValidationTypeChecker.GetAbstractValidatorTypeArgument(symbol)` -- skip if null (per D-07)
        5. Call `FluentValidationTypeChecker.GetLifetimeFromAttributes(symbol)` to get lifetime string
        6. Return `new ValidatorClassInfo(symbol, typeDecl, ctx.SemanticModel, validatedType, lifetime, ...)`
    - Chain `.Where(static x => x.HasValue).Select(static (x, _) => x!.Value)`
    - Chain `.Collect()` for batch processing in emitter

    **ValidatorRegistrationGenerator.cs:**
    - Namespace: `IoCTools.FluentValidation.CodeGeneration`
    - `internal static class ValidatorRegistrationGenerator`
    - Method `GenerateRegistrationLine(ValidatorClassInfo validator)` returning `string`:
      - Per D-08, generate TWO registration lines:
        1. `services.Add{Lifetime}<IValidator<{ValidatedTypeFQN}>, {ValidatorFQN}>();` (interface registration)
        2. `services.Add{Lifetime}<{ValidatorFQN}>();` (concrete registration)
      - Where Lifetime = Scoped/Singleton/Transient from validator info
      - Use fully-qualified type names to avoid using statement complexity
      - Prefix with `global::` for unambiguous resolution
    - Method `GeneratePartialMethodBody(ImmutableArray<ValidatorClassInfo> validators, string methodNamePrefix, string extNameSpace)` returning `string`:
      - Generate the full partial class file:
        ```csharp
        #nullable enable
        namespace {extNameSpace};

        using Microsoft.Extensions.DependencyInjection;

        public static partial class GeneratedServiceCollectionExtensions
        {
            static partial void Add{methodNamePrefix}FluentValidationServices(IServiceCollection services)
            {
                {for each validator: GenerateRegistrationLine(validator)}
            }
        }
        ```
      - Add FluentValidation namespace if needed for IValidator<T>: `using FluentValidation;`

    **ValidatorRegistrationEmitter.cs:**
    - Namespace: `IoCTools.FluentValidation.Generator`
    - `internal static class ValidatorRegistrationEmitter`
    - Method `Emit(SourceProductionContext context, ImmutableArray<ValidatorClassInfo> validators)`:
      - If no validators, return (no-op, partial method stays unimplemented)
      - Get compilation from the first validator's SemanticModel.Compilation
      - Compute `assemblyName`, `safeRootNamespace`, `extensionNamespace`, `safeAssemblyName` using EXACT same logic as RegistrationEmitter.cs lines 54-57:
        ```
        var assemblyName = compilation.AssemblyName ?? "UnknownAssembly";
        var safeRootNamespace = assemblyName.Replace("-", "_").Replace(" ", "_");
        var extensionNamespace = safeRootNamespace + ".Extensions.Generated";
        var safeAssemblyName = safeRootNamespace.Replace(".", "");
        ```
      - Call `ValidatorRegistrationGenerator.GeneratePartialMethodBody(validators, safeAssemblyName, extensionNamespace)`
      - Add source: `context.AddSource($"FluentValidation_{safeAssemblyName}.g.cs", SourceText.From(code, Encoding.UTF8))`
      - Wrap in try/catch (generator never throws per project constraint), emit IOC999-equivalent on failure

    NOTE: The emitter needs Compilation access. The pipeline must combine validators with compilation. Use the pattern from ServiceClassPipeline: combine the collected validators with `context.CompilationProvider`:
    ```csharp
    var combined = validators.Combine(context.CompilationProvider);
    context.RegisterSourceOutput(combined, (ctx, data) => {
        var (validatorArray, compilation) = data;
        ValidatorRegistrationEmitter.Emit(ctx, validatorArray, compilation);
    });
    ```
    Update EmitMethod signature to accept Compilation as a parameter instead of extracting from SemanticModel.
